## 一、执行摘要

**Pawly（宝莉）是一个 AI 原生的宠物消费与服务平台。** 用户无需注册，与 AI 客服"宝莉助手"对话即可自动建立宠物档案；平台结合宠物品种、年龄、体重等特征，从真实商品库智能推荐个性化购物方案，实现"提问 → 选方案 → 下单"的短链路购物体验，并以循证科普内容与铲屎官社区构建信任与留存。

一句话定位：**给年轻宠物主的"会问、会想、会推荐"的智能宠物管家，差异化于传统货架式宠物电商。**

当前状态：全功能 Web 产品已上线（AI 导购、多 Agent 架构、循证科普 58 篇、社区、会员额度体系、订单与地址闭环、移动端适配），正推进公司注册与微信小程序上线。

## 二、产品介绍

### 2.1 解决什么问题

| 用户痛点 | 现状 | Pawly 的解法 |
|---|---|---|
| 选品难 | 货架式电商靠用户自己搜、比、猜，新手无从下手 | AI 对话式导购：说一句"我家柴犬 2 个月"，方案直接送到面前 |
| 信息不可信 | 全网养宠内容真假混杂，AI 生成内容泛滥 | 循证科普：全部编译自 WSAVA/AAHA 等权威兽医指南并标注来源 |
| 决策孤独 | 新手铲屎官缺乏同伴经验参考 | 铲屎官社区：晒宠、好物种草、养宠求助 |
| 数据割裂 | 宠物信息散落在聊天记录和记忆里 | 宠物档案：对话中自动建档，存出生日期实时算年龄，跨设备同步 |

### 2.2 核心功能（已全部实现）

1. **AI 智能导购「宝莉助手」**
   - 对话中自动识别并记录宠物信息（品种/月龄/体重/特点），无需填表
   - 结合档案从 38 款真实商品库检索，输出 2–3 套结构化购物方案（经济型→周全型），一键采纳加购
   - 每日额度体系：游客 5 次/天，会员 30 次/天（会员核心权益之一）

2. **多 Agent 架构**（对应五层架构设计）
   - 客服主 Agent：意图路由 + 编排循环 + 输出守门
   - 知识 Agent：证据检索 → 风险分级（本地规则判定，不信任模型）→ 引用收口 → 高风险强制就医引导与转人工信号
   - 工具层：宠物档案、商品检索与排序、社区检索、订单、知识问答等 10 个工具
   - 安全设计：医疗问题不做诊断性断言、无证据保守拒答、未读取原文的来源绝不用"参考"背书

3. **循证科普（58 篇）**
   - 12 篇深度长文 + 30 篇单点短科普（300–500 字/篇）+ 16 篇日常向文章
   - 全部编译自 WSAVA、AAHA、iCatCare、ASPCA、ESCCAP、AVSAB、AAFP/ISFM、Merck 兽医手册等权威指南，文末"参考来源"可点击直达原文，列表页"✓ 循证"角标
   - 同一份内容同时喂给知识 Agent 作为检索证据库——内容资产与 AI 能力复用

4. **铲屎官社区**：瀑布流帖子（晒宠/好物/求助/日常）、发帖、点赞、详情弹窗，帖子可关联自家宠物档案

5. **电商闭环**：38 款商品（详情+规格）、购物车、收货地址簿（多地址/默认地址）、真实订单（待支付状态机预留支付接入）、全站搜索（商品+科普分词匹配）

6. **会员体系（Pawly Club）**：手机号一键登录即会员；游客期数据（档案/帖子/订单）登录时自动并入账号，跨设备同步；会员权益页展示 AI 额度进度与权益矩阵

7. **多端就绪**：桌面 Web + 移动端竖屏完整适配；前后端逻辑分离（BFF + 内网后端服务），同一套后端可直接支撑微信小程序与 App

### 2.3 技术架构

```
用户入口   Web（已上线）→ 微信小程序（开发中）→ App（规划）
接入层     Next.js BFF（会话 Cookie / 身份 / 代理转发）——公网仅暴露此层
Agent 层   客服主 Agent ⇄ 知识 Agent（DeepSeek API · Function Calling）
工具层     宠物档案 / 商品推荐 / 社区检索 / 订单 / 知识问答 / 风控转人工(信号)
数据层     PostgreSQL（用户/宠物/商品/订单/帖子/地址/AI用量）+ 循证科普知识库
```

- 后端服务只监听内网 127.0.0.1 + 内部密钥双重隔离，数据库/模型 Key 不在公网进程
- 双模式部署：自有服务器（分离模式）与 Vercel（单体模式）同一套代码
- 工程实践：结构化输出防 JSON 泄露、额度故障放行（增值能力不阻断核心服务）、多智能体代码评审流程

### 2.4 产品进度与验证

- 全链路已实测：AI 建档→推荐→加购→下单→订单查询；登录合并；社区发帖点赞；额度拦截
- 已知待办：真实支付（微信支付）、短信验证码（需公司主体申请签名）、图片上传（对象存储）、真实供应链对接

## 三、市场分析

### 3.1 市场规模与趋势

- 据《中国宠物行业白皮书》等公开行业报告，中国城镇犬猫数量已超 1.2 亿只，城镇犬猫消费市场规模约 3000 亿元人民币，且保持稳定增长；单只宠物年均消费持续上升。
- 结构性趋势：**养宠人群年轻化**（90/00 后成为主力，天然接受 AI 与线上消费）；**宠物家人化**带来"科学养宠"付费意愿；**决策线上化**（小红书式种草 + 电商购买已是主流路径）。
- AI 大模型的成熟使"对话式导购"首次具备可用性——这是传统宠物电商在其货架架构上难以复制的时间窗口。

### 3.2 竞争格局与差异化

| 竞品类型 | 代表 | 优势 | 我们的差异化 |
|---|---|---|---|
| 综合电商 | 京东/淘宝宠物频道 | 供应链、价格 | 他们是"人找货"，Pawly 是"AI 懂你的宠物后送方案上门"；且早期直接复用其供应链做联营 |
| 垂直宠物电商 | 波奇、E宠 | 品类深度 | 决策入口不同：我们以 AI 咨询与循证内容为入口，购买是咨询的自然延伸 |
| 内容社区 | 小红书宠物区 | 流量、UGC | 内容不可信度高、种草到购买链路断裂；Pawly 循证内容+站内闭环 |
| AI 问诊类 | 各类宠物问诊 App | 医疗垂直 | 我们不做诊断（合规红线），做"养护咨询+消费决策"，就医场景导流给持证机构 |

**核心壁垒**：AI 对话沉淀的结构化宠物档案数据。档案 → 更准的推荐 → 更高转化与复购 → 更多数据，形成数据飞轮；这是货架式电商补不出来的资产。

## 四、商业模式

### 4.1 收入结构（三阶段）

| 阶段 | 模式 | 说明 |
|---|---|---|
| 第 1 阶段（0–6 月） | **导购 + 联盟佣金** | 对接京东联盟/淘宝联盟等分佣，零库存零食品资质压力，专注验证 AI 导购的转化率与复购率 |
| 第 2 阶段（6–18 月） | **自营精选 + 会员订阅** | 用转化数据反向选品，引入高毛利自营品类；Pawly Club 付费会员（AI 额度、专属价、体检等权益，现有免费会员体系平滑升级） |
| 第 3 阶段（18 月+） | **数据驱动的平台化** | 宠物档案驱动个性化推荐与自有供应链；开放品牌方投放与联名 |

### 4.2 单位经济模型（假设，待数据验证）

- 获客：微信生态社群裂变 + 循证内容自然流量，目标 CAC 显著低于投流型电商
- 转化：AI 方案"一键采纳"缩短决策链路，核心指标为"咨询→下单转化率"与"档案用户 90 天复购率"
- 留存：宠物档案与会员额度构成切换成本；宠物消费天然高频（主粮 1–2 月一购）

### 4.3 冷启动打法

微信小程序为主战场：宠物主社群运营 + KOC 测评种草 + 视频号内容导流；"AI 免费帮你家毛孩子挑粮"作为钩子产品，转化到档案与首单。

## 九、风险与应对

| 风险 | 应对 |
|---|---|
| 生成式 AI 备案周期不可控 | 第 1 周提交；备用方案为"人工客服+规则引擎"低配版先上线，备案通过后灰度放开 AI |
| 大模型单点依赖 | 接入备用模型（Qwen/GLM 等）做容灾；成本上限告警 |
| AI 医疗内容责任 | 多层护栏：本地规则风险分级、无证据拒答、强制就医引导、循证来源标注、免责声明 |
| 供应链与履约 | 早期联营把库存与资质压力转移给持证上游；自营前建立质检与售后流程 |
| 巨头复制 | 以档案数据飞轮与循证内容心智建立先发壁垒；小程序生态快速渗透 |
| 团队规模 | 法务/代办类工作外包，团队聚焦产品与增长 |
