# Pawly 自有服务器部署指南

> 适用场景：从 Vercel 迁到自己的 Linux 服务器（如 Pawly1，端口已放行 22、28888-28898）。
> 应用跑在 **28888** 端口，浏览器访问 `http://<公网IP>:28888`。

## 0. 前置说明

- **数据库二选一**：
  - **A. 继续用 Neon**（最省事）：`.env` 里的 `DATABASE_URL` 不变。缺点是国内服务器连境外 Neon 延迟高（每次查询 200ms+），演示可接受，正式运营不建议。
  - **B. 服务器本地装 PostgreSQL**（推荐）：延迟 <1ms，数据自主可控，见第 2 步。
- 短信未接入前，登录验证码为**演示模式**（验证码直接显示在登录框），接入阿里云/腾讯云短信后自动切换真实发送。
- ⚠️ 目前是 `http://IP:端口` 裸奔访问。绑定域名 + HTTPS 之前不要投放给真实用户（cookie 明文传输）；域名备案后用 Nginx + Let's Encrypt 补上。

## 1. 安装 Node.js 20 与 PM2（root 执行）

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -   # Debian/Ubuntu
apt-get install -y nodejs git
# CentOS/阿里云Linux 用: curl -fsSL https://rpm.nodesource.com/setup_20.x | bash - && yum install -y nodejs git
npm install -g pm2
```

## 2.（可选，推荐）本地 PostgreSQL

```bash
apt-get install -y postgresql
sudo -u postgres psql -c "CREATE USER pawly WITH PASSWORD '换成强密码';"
sudo -u postgres psql -c "CREATE DATABASE pawly OWNER pawly;"
```
连接串：`postgresql://pawly:换成强密码@localhost:5432/pawly`
（数据库只监听 localhost，不要对公网开放 5432。）

## 3. 拉代码并配置环境变量

```bash
cd /opt
git clone https://github.com/Celveen/Pawly.git pawly
cd pawly

cat > .env <<'EOF'
DATABASE_URL=postgresql://pawly:换成强密码@localhost:5432/pawly
EOF

cat > .env.local <<'EOF'
DEEPSEEK_API_KEY=你的key
PAWLY_MODEL=deepseek-v4-flash
DEEPSEEK_BASE_URL=https://api.deepseek.com/chat/completions
AUTH_SECRET=用 openssl rand -hex 32 生成一个随机串填这里
EOF
chmod 600 .env .env.local
```

## 4. 构建并启动

```bash
npm install
npx prisma db push          # 同步全部数据表（User/Pet/Product/Order/Post/PostLike/PhoneCode）
npm run build
PORT=28888 pm2 start npm --name pawly -- start
pm2 save && pm2 startup     # 开机自启（按输出提示再执行一条命令）
```

验证：`curl -s http://localhost:28888/api/products | head -c 200`，浏览器访问 `http://公网IP:28888`。

## 5. 日常更新

```bash
cd /opt/pawly
git pull
npm install
npx prisma db push          # schema 有变更时
npm run build
pm2 restart pawly
```

## 6. 数据备份（本地 Postgres 时必做）

```bash
# 每天凌晨 3 点自动备份，保留 14 天（crontab -e 加入）：
0 3 * * * pg_dump -U pawly pawly | gzip > /opt/backups/pawly-$(date +\%F).sql.gz && find /opt/backups -name '*.sql.gz' -mtime +14 -delete
```

## 7. 后续（有域名并完成 ICP 备案后）

1. 域名解析到公网 IP；
2. 装 Nginx 反代 28888 → 443，certbot 签发免费证书；
3. `lib/session.ts` 的 cookie 加 `secure: true`；
4. 防火墙收窄：对公网只留 80/443（22 限制来源 IP）。
